# 📚 What I Learned

## What is Observability?

Observability is the ability to understand the internal state of a system by analyzing the data it generates.

Unlike traditional monitoring, observability helps engineers investigate **why** an issue occurred instead of only notifying that something is wrong.

### Monitoring vs Observability

| Monitoring | Observability |
|------------|---------------|
| Detects failures | Helps investigate failures |
| Based on predefined alerts | Supports deep troubleshooting |
| Answers **What happened?** | Answers **Why did it happen?** |
| Reactive | Proactive |

---

# 🔍 Three Pillars of Observability

## 1️⃣ Metrics

Metrics are numerical values collected over time.

### Examples

- CPU Usage
- Memory Usage
- Disk Utilization
- HTTP Requests
- Error Rate

### Popular Tools

- Prometheus
- Datadog
- CloudWatch

---

## 2️⃣ Logs

Logs are timestamped records generated by applications and systems.

### Examples

- Application logs
- Error logs
- Access logs
- Authentication logs

### Popular Tools

- ELK Stack
- Loki
- Fluentd

---

## 3️⃣ Traces

Traces show how a single request travels through multiple services.

Useful in Microservices Architecture.

### Popular Tools

- Jaeger
- Zipkin
- OpenTelemetry

---

# Why DevOps Engineers Need All Three

| Pillar | Helps You Know |
|---------|----------------|
| Metrics | What is happening |
| Logs | Why it happened |
| Traces | Where it happened |

Example:

- CPU becomes 100% → Metrics
- Database timeout → Logs
- Payment service delay → Trace

```yaml

  scrape_interval: 15s
  evaluation_interval: 15s

scrape_configs:
  - job_name: "prometheus"
    static_configs:
      - targets:
          - localhost:9090
```

---

# Step 3 - Create docker-compose.yml

```yaml
services:
  prometheus:
    image: prom/prometheus:latest
    container_name: prometheus

    ports:
      - "9090:9090"

    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
      - prometheus_data:/prometheus

    command:
      - '--config.file=/etc/prometheus/prometheus.yml'

    restart: unless-stopped

volumes:
  prometheus_data:
```

# Understanding Prometheus Concepts

## Scrape Targets

Prometheus collects metrics by periodically pulling data from configured targets.

This is called the **Pull Model**.

---

## Metric Types

### Counter

Only increases.

Examples

- Total Requests
- Total Errors
- Login Attempts

Example Metric

```
http_requests_total
```

---

### Gauge

Can increase or decrease.

Examples

- CPU Usage
- Memory Usage
- Active Connections

Example

```
process_resident_memory_bytes
```

---

### Histogram

Stores observations into buckets.

Used for

- Request Duration
- API Response Time

---

### Summary

Calculates percentiles.

Example

- 95th Percentile Response Time

---

## Labels

Labels add extra information to metrics.

Example

```
http_requests_total{
method="GET",
status="200"
}
```

---

## Time Series

A Time Series is

```
Metric Name
+
Labels
+
Timestamp
```

# Key Interview Questions

### What is Observability?

Observability is the ability to understand the internal state of a system using Metrics, Logs, and Traces.

---

### What are the Three Pillars?

- Metrics
- Logs
- Traces

---

### What is Prometheus?

Prometheus is an open-source monitoring and alerting toolkit that collects metrics using a pull-based model.

---

### What is PromQL?

PromQL (Prometheus Query Language) is used to query, filter, aggregate, and analyze Prometheus metrics.

---

### Difference Between Counter and Gauge?

| Counter | Gauge |
|----------|-------|
| Only increases | Can increase or decrease |
| Used for requests/errors | Used for CPU, Memory |

---

### What is Scraping?

Prometheus periodically pulls metrics from configured targets.

---

### What is the `up` Metric?

Returns:

- **1** → Target is reachable
- **0** → Target is down

---

### Why use `rate()`?

Counters always increase. `rate()` calculates the per-second increase over a specified time range, making counter metrics meaningful for monitoring.
